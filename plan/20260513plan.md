# alpha2rescore Development Plan

## Summary

Build `alpha2rescore` as an installable Python package under `alpha2rescore/`, with the first implementation step writing this plan to `alpha2rescore/plan/20260513plan.md`.

The v1 goal is a fast, incremental Percolator-input generator for `ms2pin.parquet/<idn>.pin.parquet` plus `mzDuck/<idn>.mgf.parquet`, using:

- AlphaPept predictions from PostgreSQL when available.
- Local AlphaPept prediction cache for missing precursors.
- DeepLC RT features by default, with one small reusable calibration file per idn.
- No edits to `DeepLC`, `IM2Deep`, `ms2pip`, or `ms2rescore`.

## Key Changes

- Package layout:
  - `alpha2rescore/pyproject.toml`
  - `alpha2rescore/src/alpha2rescore/`
  - CLI entry point: `alpha2rescore`
  - install command: `pip install -e /data/p/ms2rescore/alpha2rescore`

- CLI:
  - `alpha2rescore build --pin-parquet ... --mgf-parquet ... --out-dir ...`
  - `alpha2rescore build --pin-dir ... --mgf-dir ... --idn 1554451 --out-dir ...`
  - rich `argparse --help` with examples, env defaults, PostgreSQL defaults, cache behavior, and PXD010154 examples.

- Input handling:
  - Read Comet PIN parquet columns directly.
  - Read `mzDuck/*.mgf.parquet` columns `scan_number`, `rt_seconds`, `mz_array`, `intensity_array`.
  - Match PSMs to spectra by `ScanNr -> scan_number`; do not depend on MGF text export.

- Peptide to PostgreSQL mapping:
  - Do not use `Proteins`.
  - Parse `Peptide` normalized `[U:id]` strings into `pep_seq` plus `var_mod_sites_unimod`.
  - Use `Label` to choose target vs decoy tables.
  - Bulk lookup `variant_id`, then `precursor_id = variant_id * 10 + charge`.
  - Query `protein_hs.protinsight_trypsin_variant_alphapep` or decoy equivalent for `peak_code[]`, `mz[]`, `intensity[]`.

- Feature generation:
  - Reconstruct AlphaPept predicted b/y arrays from PostgreSQL `fragment_peak_codes`.
  - Match observed mzDuck peaks by m/z tolerance.
  - Emit the existing 71 MS2PIP-style spectral feature names used in prior AlphaPeptMS2 tests.
  - Add DeepLC features by default: `observed_retention_time`, `predicted_retention_time`, `rt_diff`, and best-per-peptide variants.

- Incremental cache:
  - Per idn cache directory under output, storing lookup results, AlphaPept feature parquet, missing local predictions, DeepLC feature parquet, DeepLC calibration pickle, and a manifest.
  - Stable PSM key: `idn + SpecId + Peptide + charge + Label`.
  - Recompute only missing or changed keys; rewrite final PIN in original row order.

## Public Interfaces

- Python API:
  - `build_pin(config: Alpha2RescoreConfig) -> BuildResult`
  - `build_features(pin_parquet, mgf_parquet, config) -> pandas.DataFrame`
  - `load_deeplc_calibration(idn, cache_dir)` and `save_deeplc_calibration(...)`

- Default subprocess interpreters:
  - Main/DeepLC: configurable, defaulting to project ms2rescore Python.
  - AlphaPept missing prediction: `/data/p/anaconda3/envs/alphabase/bin/python`
  - PostgreSQL helper: `/data/p/anaconda3/bin/python`

- Default missing-prediction policy:
  - `local-cache-only`; never write missing predictions back to PostgreSQL in v1.

## Test Plan

- Unit tests:
  - Parse `[U:id]` peptides, including residue mods and N-terminal `[U:1]-`.
  - Validate charge extraction from `Charge1..Charge6`.
  - Validate mzDuck scan matching and feature array reconstruction.
  - Validate PIN writer preserves original rows and inserts feature columns before `Peptide`/`Proteins`.

- Integration smoke tests:
  - Use `1554451.pin.parquet` and matching `1554451.mgf.parquet`.
  - Build first 500 to 2000 PSMs into `/XCLabServer002_fastIO/ms2rescore-test/alpha2rescore/`.
  - Confirm PostgreSQL lookup hit rate, local missing count, row retention, finite feature values, and valid gzipped PIN output.
  - Run twice and verify the second run reuses caches and avoids recomputing unchanged features.
  - Confirm `alpha2rescore --help` and subcommand help print detailed usage.

## Assumptions

- Default output is one AlphaPeptMS2 + DeepLC Percolator PIN per idn.
- Missing PostgreSQL AlphaPept predictions are predicted locally and cached only locally.
- `Peptide`, not `Proteins`, is the source of peptide identity.
- PXD010154 optimized path targets `ms2pin.parquet` plus `mzDuck/*.mgf.parquet`; legacy MGF/mzML support is not required for v1.
